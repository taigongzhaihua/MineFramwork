# mine.io 模块 API 总览

## 概述

**模块**：`mine.io`  
**头文件**：`<mine/io/Io.h>` （统一包含头）  
**命名空间**：`mine::io`

`mine.io` 是 MineFramework 的 L0 基础模块,提供跨平台的文件系统访问、文件读写、内存映射和流式 I/O 能力。

---

## 模块组成

| 类型 | 头文件 | 用途 |
|------|--------|------|
| `Path` | `<mine/io/Path.h>` | 路径字符串封装与操作 |
| `File` | `<mine/io/File.h>` | 文件句柄与读写操作 |
| `FileSystem` | `<mine/io/FileSystem.h>` | 文件系统查询与操作 |
| `MemMap` | `<mine/io/MemMap.h>` | 内存映射文件视图 |
| `PipeStream` | `<mine/io/PipeStream.h>` | 缓冲流式读写 |

## 快速索引

### Path — 路径操作
- **用途**：跨平台路径字符串封装
- **特性**：SBO 优化、UTF-8 编码、规范化分隔符
- **核心 API**：
  - 构造：`Path(const char*)` / `Path(StringView)`
  - 组件查询：`filename()` / `extension()` / `parent_path()`
  - 拼接：`operator/` / `join()`
  - 修改：`replace_extension()` / `replace_filename()`
- **详细文档**：[01-Path.md](01-Path.md)

**示例**：
```cpp
Path p = Path::from("/home/user/report.pdf");
Path parent = p.parent_path();           // "/home/user"
StringView ext = p.extension();          // ".pdf"
Path full = Path::from("/usr") / "local" / "bin";  // "/usr/local/bin"
```

---

### File — 文件读写
- **用途**：文件句柄管理与 I/O 操作
- **特性**：RAII 管理、显式错误处理、移动语义、随机访问
- **核心 API**：
  - 打开：`open(path, mode)` / `open_with_access()`
  - 读取：`read()` / `read_all()` / `read_exact()`
  - 写入：`write()` / `write_all()` / `flush()`
  - 定位：`seek()` / `tell()`
  - 大小：`size()` / `set_size()`
- **详细文档**：[02-File.md](02-File.md)

**示例**：
```cpp
// 读取配置文件
auto file = File::open("config.json", FileMode::Read).value();
auto content = file.read_all();

// 写入日志
auto log = File::open("app.log", FileMode::Write | FileMode::Create | FileMode::Append).value();
log.write_all("Log entry\n");
log.flush();
```

---

### FileSystem — 文件系统操作
- **用途**：文件系统查询与操作（无状态函数集合）
- **特性**：静态方法、原子操作、符号链接感知、目录遍历
- **核心 API**：
  - 查询：`exists()` / `is_file()` / `is_directory()` / `file_size()`
  - 创建：`create_directory()` / `create_directories()`
  - 删除：`remove()` / `remove_all()`
  - 复制/移动：`copy()` / `rename()`
  - 遍历：`enumerate_directory(callback)`
  - 路径工具：`absolute()` / `canonical()` / `current_path()`
- **详细文档**：[03-FileSystem.md](03-FileSystem.md)

**示例**：
```cpp
// 检查文件存在性
if (FileSystem::exists(Path::from("config.json"))) {
    // ...
}

// 遍历目录
FileSystem::enumerate_directory(Path::from("/data"), [](const DirectoryEntry& entry) {
    printf("%s\n", entry.name.c_str());
    return true;  // 继续枚举
});

// 递归创建目录
FileSystem::create_directories(Path::from("/a/b/c"));
```

---

### MemMap — 内存映射
- **用途**：零拷贝文件访问,直接映射到进程地址空间
- **特性**：零拷贝、RAII 管理、多种模式（只读/读写/COW）
- **核心 API**：
  - 打开：`map_file(path, mode)`
  - 访问：`data()` / `data_mut()` / `size()` / `as_span()`
  - 同步：`flush()`
- **详细文档**：[04-MemMap.md](04-MemMap.md)

**示例**：
```cpp
// 只读映射大文件
auto map = MemMap::map_file("large_data.bin", MemMap::MapMode::ReadOnly).value();
Span<const uint8_t> data = map.data();

// 解析二进制头
const auto* header = reinterpret_cast<const BinaryHeader*>(data.data());

// 读写映射
auto rw_map = MemMap::map_file("cache.dat", MemMap::MapMode::ReadWrite).value();
Span<uint8_t> writable = rw_map.data_mut();
writable[0] = 0xFF;
rw_map.flush();  // 同步到磁盘
```

---

### PipeStream — 缓冲流
- **用途**：带缓冲的流式读写,适合频繁小 I/O
- **特性**：8KB 双缓冲、行处理、自动刷新、RAII 管理
- **核心 API**：
  - 打开：`open_read()` / `open_write()` / `open_append()`
  - 读取：`read()` / `read_line()`
  - 写入：`write()` / `write_line()` / `flush()`
- **详细文档**：[05-PipeStream.md](05-PipeStream.md)

**示例**：
```cpp
// 逐行读取配置文件
auto stream = PipeStream::open_read("config.ini").value();
String line;
while (stream.read_line(line).value()) {
    process_line(line);
}

// 追加日志
auto log = PipeStream::open_append("app.log").value();
log.write_line("Application started");
log.flush();
```

## 使用决策树

### 如何选择合适的类型？

```
需要操作路径字符串?
├─ 是 → Path
└─ 否
    ├─ 需要文件系统操作（创建/删除/查询）?
    │  └─ 是 → FileSystem
    └─ 否 → 需要读写文件?
        ├─ 大文件随机访问?
        │  └─ 是 → MemMap
        └─ 否
            ├─ 逐行处理或频繁小 I/O?
            │  └─ 是 → PipeStream
            └─ 大块 I/O 或随机定位?
               └─ 是 → File
```

### 性能对比

| 场景 | 推荐类型 | 理由 |
|------|----------|------|
| 小文件一次性读取 | `File::read_all()` | 简单高效,一次系统调用 |
| 大文件顺序读取 | `PipeStream` | 8KB 缓冲减少系统调用 |
| 逐行处理（日志/配置） | `PipeStream::read_line()` | 自动处理行边界 |
| 随机访问（索引/数据库） | `MemMap` | 零拷贝,按需分页加载 |
| 频繁小写入 | `PipeStream` | 缓冲聚合写入 |
| 大块写入 | `File::write()` | 直接写入,无缓冲开销 |

## 设计原则

### 1. 无异常错误处理

所有可能失败的操作返回 `Result<T>` 或 `Status`,编译器强制错误检查：

```cpp
// ✅ 显式检查
auto result = File::open("data.txt", FileMode::Read);
if (!result) {
    handle_error(result.error());
    return;
}
File file = std::move(result.value());

// ✅ MINE_TRY 宏传播错误
Result<void> process() {
    File file = MINE_TRY(File::open("data.txt", FileMode::Read));
    // ...
}
```

### 2. RAII 生命周期管理

资源（文件句柄/内存映射）通过 RAII 自动管理,析构时自动释放：

```cpp
{
    File file = File::open("temp.txt", FileMode::DefaultWrite).value();
    file.write_all("data");
    // 析构时自动关闭文件
}
```

### 3. 移动语义

文件句柄等资源类型仅移动,不可拷贝,避免意外的资源复制：

```cpp
File file1 = File::open("data.txt", FileMode::Read).value();
File file2 = std::move(file1);  // ✅ 移动
// File file3 = file2;          // ❌ 编译错误：不可拷贝
```

### 4. UTF-8 统一编码

所有路径字符串使用 UTF-8 编码,Windows API 调用时自动转 UTF-16：

```cpp
Path p = Path::from(u8"文档/测试.txt");  // UTF-8 路径
File file = File::open(p, FileMode::Read).value();  // 内部自动转 UTF-16（Windows）
```

### 5. 平台透明

公共 API 跨平台一致,平台差异在内部处理：

```cpp
// 相同代码在 Windows/Linux/macOS 上都能运行
Path p = Path::from("/home/user/file.txt");
if (FileSystem::exists(p)) {
    File file = File::open(p, FileMode::Read).value();
}
```

## 典型使用模式

### 模式 1：配置文件读取

```cpp
Result<String> read_config(const Path& config_path) {
    // 检查文件是否存在
    if (!FileSystem::exists(config_path)) {
        return err_tag, Status::not_found("配置文件不存在");
    }
    
    // 打开文件
    File file = MINE_TRY(File::open(config_path, FileMode::Read));
    
    // 获取文件大小
    size_t size = MINE_TRY(file.size());
    
    // 分配缓冲区并读取
    Vector<char> buffer(size);
    MINE_TRY(file.read_all(Span<char>(buffer.data(), size)));
    
    return ok_tag, String(buffer.data(), size);
}
```

### 模式 2：日志追加

```cpp
class Logger {
    PipeStream stream_;
    
public:
    Result<void> init(const Path& log_path) {
        stream_ = MINE_TRY(PipeStream::open_append(log_path));
        return ok_tag;
    }
    
    void log(StringView level, StringView message) {
        auto now = std::time(nullptr);
        char timestamp[32];
        std::strftime(timestamp, sizeof(timestamp), "%Y-%m-%d %H:%M:%S", std::localtime(&now));
        
        String line = String::format("[%s] [%s] %s", 
                                      timestamp, 
                                      level.data(), 
                                      message.data());
        stream_.write_line(line);
        stream_.flush();  // 立即持久化
    }
};
```

### 模式 3：目录遍历

```cpp
void walk_directory_tree(const Path& root, std::function<void(const Path&)> callback) {
    FileSystem::enumerate_directory(root, [&](const FileSystem::DirectoryEntry& entry) {
        Path full_path = root / entry.name;
        callback(full_path);
        
        if (entry.is_directory) {
            walk_directory_tree(full_path, callback);  // 递归遍历
        }
        
        return true;  // 继续枚举
    });
}

// 使用
walk_directory_tree(Path::from("/data"), [](const Path& p) {
    printf("Found: %s\n", p.c_str());
});
```

### 模式 4：大文件索引搜索

```cpp
Result<size_t> find_pattern_in_file(const Path& file_path, const uint8_t* pattern, size_t pattern_len) {
    // 内存映射文件
    MemMap map = MINE_TRY(MemMap::map_file(file_path, MemMap::MapMode::ReadOnly));
    Span<const uint8_t> data = map.data();
    
    // 搜索模式（Boyer-Moore 等算法）
    for (size_t i = 0; i <= data.size() - pattern_len; ++i) {
        if (std::memcmp(data.data() + i, pattern, pattern_len) == 0) {
            return ok_tag, i;  // 返回偏移
        }
    }
    
    return err_tag, Status::not_found("未找到模式");
}
```

### 模式 5：原子文件写入

```cpp
Result<void> write_file_atomic(const Path& target_path, Span<const uint8_t> data) {
    // 写入临时文件
    Path temp_path = target_path.string() + ".tmp";
    {
        File temp = MINE_TRY(File::open(temp_path, FileMode::DefaultWrite));
        MINE_TRY(temp.write_all(data));
        MINE_TRY(temp.flush());
    }  // 确保文件关闭
    
    // 原子重命名（覆盖原文件）
    MINE_TRY(FileSystem::rename(temp_path, target_path));
    
    return ok_tag;
}
```

## 错误处理策略

### 错误码分类

| 错误码 | 含义 | 常见于 |
|--------|------|--------|
| `NotFound` | 文件/目录不存在 | `File::open` / `FileSystem::stat` |
| `AlreadyExists` | 文件/目录已存在 | `File::open(Exclusive)` / `FileSystem::create_directory` |
| `PermissionDenied` | 权限不足 | `File::open` / `FileSystem::remove` |
| `InvalidArgument` | 参数非法 | `Path` 构造 / `File::seek` |
| `IoError` | 通用 I/O 错误 | 磁盘满 / 硬件故障 |
| `IsDirectory` | 路径是目录而非文件 | `File::open` |
| `NotDirectory` | 路径是文件而非目录 | `FileSystem::enumerate_directory` |

### 错误处理模板

```cpp
// 模板 1：Result 显式检查
auto result = File::open("data.txt", FileMode::Read);
if (!result) {
    switch (result.error().code()) {
        case StatusCode::NotFound:
            // 文件不存在 → 创建默认文件
            break;
        case StatusCode::PermissionDenied:
            // 权限不足 → 提示用户
            break;
        default:
            // 其他错误 → 失败返回
            return result.error();
    }
}

// 模板 2：MINE_TRY 传播错误
Result<void> process_file(const Path& path) {
    File file = MINE_TRY(File::open(path, FileMode::Read));
    // ...
    return ok_tag;
}

// 模板 3：value_or 提供回退
File file = File::open("config.json", FileMode::Read)
    .value_or(File());  // 打开失败返回空句柄
```

## 性能优化建议

### 1. 选择合适的 I/O 方式

| 场景 | 不推荐 | 推荐 |
|------|--------|------|
| 频繁小写入 | `File::write()` 每次 1 行 | `PipeStream::write_line()` 缓冲 |
| 大文件随机访问 | `File::seek() + read()` | `MemMap` 内存映射 |
| 逐行处理 | `File::read() + 手动分割` | `PipeStream::read_line()` |

### 2. 减少系统调用

```cpp
// ❌ 低效：每次 1 字节
for (char c : data) {
    file.write(Span<const char>(&c, 1));  // 1M 次系统调用
}

// ✅ 高效：批量写入
file.write_all(data);  // 1 次系统调用
```

### 3. 利用 SBO 优化

```cpp
// ✅ 短路径无堆分配（≤31 字节）
Path p1("config.json");  // 栈上分配

// ⚠️ 长路径堆分配
Path p2("/very/long/path/to/some/deep/directory/file.txt");

// ✅ 缓存路径避免重复构造
static Path cache_dir = Path::from("/app/cache");
```

### 4. 避免不必要的拷贝

```cpp
// ❌ 多次拷贝
Path p1 = Path::from("/data/file.txt");
Path p2 = p1;  // 拷贝
Path p3 = p2;  // 再次拷贝

// ✅ 使用移动语义
Path p1 = Path::from("/data/file.txt");
Path p2 = std::move(p1);  // 移动,无拷贝
```

## 已知限制

| 限制 | 影响 | 规划 |
|------|------|------|
| 仅支持 Windows | 其他平台暂不可用 | M0.2 添加 POSIX 实现 |
| 无异步 I/O | 阻塞式调用可能卡主线程 | M0.3 集成 `mine.async` |
| 无流式压缩 | 不支持 gzip/zstd 透明解压 | M1.0 添加压缩流装饰器 |
| 无网络流 | 不支持 HTTP/FTP | M1.0 集成 `mine.net` |
| 无文件监视 | 不支持目录变更通知 | M1.1 添加 `FileWatcher` |
